Feature Pipeline

Datasets & Collection

  • AMD: 24,553 malware samples, 71 families (2010–2016).
  • DREBIN: 5,560 applications, 179 malware families.[file:134]
  • AndroZoo: Benign APKs collected via az tool with size and virus-score filters.[file:134]

Static Feature Extraction

  • Permissions, intents, hardware/software features.
  • Restricted API calls and libraries (e.g., de>com.google.android.*).
  • URL domains and network endpoints grouped into behavioral categories (C2, ad networks, tracking, etc.).[file:134]

Cardinality Reduction Strategy

  • Work on a subset of ~1,000 APKs for manageable vector size.
  • Truncate and categorize high-cardinality APIs, libraries, and URLs into coarse groups.
  • Remove underused features (below threshold) and features present in nearly every APK.[file:134]

This reduces millions of possible features down to a few thousand, making dense vectorization and model training feasible on local hardware.

Models & Approaches

CNN Prototype

Initial plan: treat APK vectors as 1D signals and train a shallow CNN for binary classification (benign vs malicious). CNNs performed well on reduced vectors but were prone to overfitting on sparse, high-dimensional data.[file:134]

1D convolutions Static vectors Overfitting risk

LinearSVC (SVM)

Linear Support Vector Machine provides a strong baseline for high-dimensional binary vectors. It achieved around 85% accuracy, but lagged behind boosted trees on this feature space.[file:134]

High-dimensional Linear classifier ≈ 85% accuracy

LightGBM (GBDT)

Gradient Boosted Decision Trees via LightGBM handled the reduced feature set best, reaching ≈93% accuracy with strong precision and recall. Histogram-based splits and regularization mitigated overfitting.[file:134]

Gradient Boosting Histogram splits Best overall performance

Key Findings

Challenges

  • ~125k+ static features versus ~14k samples → extreme feature-to-sample imbalance.[file:134]
  • Memory constraints when building fully dense vectors at original dimensionality.
  • Rare single-occurrence features and duplicated library indicators increased noise.[file:134]

Lessons Learned

  • Cardinality reduction and grouping are mandatory before deep learning on APK vectors.
  • CNNs struggle when most features are sparse and “equally important”.[file:134]
  • Classical models (SVM, LightGBM) remain strong baselines for static malware detection.[file:134]

Outcome

  • End-to-end pipeline: APK collection → static feature extraction → categorical reduction → vectorization → model training and evaluation.
  • Best-performing configuration: reduced feature set + LightGBM with tuned hyperparameters.[file:134]
  • Supporting CNN experiments illustrate trade-offs between expressiveness and overfitting on sparse inputs.[file:134]

Future work could integrate dynamic features, explore federated learning, or pair static vectors with LLM-based code summarization for richer semantic signals.

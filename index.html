Feature Pipeline

Datasets & collection

  • AMD: 24,553 malware samples across 71 families (2010–2016).[web:10]
  • DREBIN: Around 5,500 apps with labeled malware families.[web:17]
  • AndroZoo: Benign APKs gathered with size and virus‑score filters.[web:20]

Static feature extraction

  • Permissions, intents, and hardware/software features from the manifest.[web:17][web:26]
  • Restricted API calls and libraries from the code.
  • URL domains and endpoints grouped into categories such as C2, ad networks, and tracking.[web:12][web:23]

Feature reduction

  • Start with a subset of ~1,000 APKs to make vector sizes manageable.
  • Group high‑cardinality APIs, libraries, and URLs into broader categories.
  • Drop features that are almost never used or appear in nearly every app.

This turns millions of possible features into a few thousand, so dense vectors and model training fit on local hardware.[web:11][web:24]

Models & Approaches

CNN prototype

The project tests a simple 1D CNN on APK feature vectors for binary classification (benign vs. malicious). It works on the reduced vectors but tends to overfit when the space is very large and sparse.[web:13][web:14]

1D convolutions Static vectors Overfitting risk

LinearSVC (SVM)

A linear SVM serves as a baseline for the high‑dimensional binary vectors. In this project it reaches around 85% accuracy but falls short of boosted tree models.[web:21][web:23]

High‑dimensional Linear model ≈ 85% accuracy

LightGBM

LightGBM, a gradient boosting tree model, handles the reduced feature set well and reaches about 93% accuracy with strong precision and recall.[web:18][web:24] Regularization and histogram splits help control overfitting.

Key Findings

Challenges

  • ~125k+ static features versus ~14k samples, so the feature space is much larger than the dataset.[web:11]
  • Dense vectors at the original size quickly hit memory limits on standard hardware.
  • Many very rare features and duplicated library indicators add noise instead of signal.[web:26]

What worked

  • Reducing feature count and grouping related features is basically required before training deep models on APK vectors.[web:13][web:14]
  • CNNs have trouble when most features are sparse and roughly equal in importance.
  • Classical models like SVM and LightGBM remain strong baselines for static Android malware detection.[web:18][web:24][web:27]

Outcome

  • Built an end‑to‑end pipeline: collect APKs → extract static features → reduce and group features → build vectors → train and evaluate models.[web:11][web:17]
  • Best run uses the reduced feature set with a tuned LightGBM model.[web:18][web:24]
  • CNN experiments show the trade‑off between model capacity and overfitting on sparse data.[web:13][web:14]

Future work could add dynamic analysis, try federated setups, or combine static vectors with code‑aware models for richer signals.[web:15][web:21]
